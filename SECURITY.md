# Security Policy

## Supported Versions

We actively support and patch the latest major version of FrankUI.

| Version | Supported          |
| ------- | ------------------ |
| 1.x.x   | :white_check_mark: |
| < 1.0.0 | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability in FrankUI, please report it privately using GitHub's built-in **Private Vulnerability Reporting** feature:

1. Navigate to the main page of the repository on GitHub.
2. Under the repository name, click the **Security** tab.
3. In the left sidebar, click **Advisories**.
4. Click **Report a vulnerability** to open the reporting form.

We will review your submission and release a patch as quickly as possible. Thank you for helping keep FrankUI secure!

## A Note on Galactic Credits (Bug Bounties)

It is an important and popular fact that while finding flaws in software can be immensely satisfying, paying people actual, spendable currency to do so is a concept that has made a lot of software creators very poor and is widely regarded as a bad move.

Therefore, FrankUI does **not** operate a bug bounty program.

If you find a security vulnerability, we will be deeply, profoundly, and existentially grateful. We might even write your name in a very nice, shiny font in our release notes. But we will not be handing over any dollars, pounds, euros, or Altarian Dollars. You do it for the glory of the universe, and perhaps a warm feeling of cosmic satisfaction. 

And above all: **Don't Panic.**
